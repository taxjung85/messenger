# UI/CSS 외주 명세서 — 카카오톡 채널 AI 사이드바 크롬 확장

> **외주 대상**: Antigravity
> **작성일**: 2026-03-10
> **범위**: 모든 UI 컴포넌트의 디자인/CSS (로직/JS 제외)

---

## 1. 프로젝트 개요

카카오톡 채널 관리자센터(business.kakao.com)에 오른쪽 사이드바를 삽입하는 크롬 확장 프로그램.
세무사무실 직원이 고객 상담 시 AI 추천 답변, To-Do 관리, 반복 일정 관리, 급여 데이터 관리, 템플릿 등을 사용.

**타겟 URL**: `https://business.kakao.com/*`
**사이드바 폭**: 300px 고정 (반응형 아님)

## 3. 전체 레이아웃

```
┌──────────────────────────────────────┬──────────────┐
│                                      │              │
│      카카오톡 채널 관리자센터          │   사이드바    │
│      (기존 페이지)                    │   300px     │
│                                      │   fixed     │
│                                      │   right:0   │
│                                      │              │
└──────────────────────────────────────┴──────────────┘
```

- 사이드바: `position: fixed`, `right: 0`, `height: 100vh`, `width: 300px`
- 토글 시 본문 `width: calc(100vw - 300px)`으로 줄어듦
- z-index: `999999`

### 레이아웃 요구사항
- `.ai-sidebar-body`는 `display: flex; flex-direction: column` 구조
- 반복 일정 섹션(`#ai-recurring-section`)은 `margin-top: auto`로 하단에 위치
- 반복 일정 목록(`#ai-recurring-list`)은 `max-height: 30vh`로 제한 (스크롤)
- todo가 적어도 반복 일정이 화면 절반 이상 올라오면 안 됨

### 4-C. To-Do 카드 (JS 동적 생성)

```
┌─────────────────────────────────────────────┐
│ ⠿ ☐ 메일이 오지 않음                    [✕] │  ← 1줄: 핸들+체크+내용+삭제
│    [정정교▼] · 채팅 49370873... · 2026.3.10 │  ← 2줄: 담당자+채팅ID+날짜
└─────────────────────────────────────────────┘
```

- 클래스: `.ai-recommendation-card`
- 드래그 가능 (⠿ 핸들)
- 체크박스: accent-color `#4f46e5`
- 내용 더블클릭 → 인라인 input 편집 (`.ai-todo-content`)
- 완료 시: 텍스트 `line-through`, 색상 `#9ca3af`
- 호버: 미세 lift + 왼쪽 그라디언트 바 (4px)
- 드래그 중: opacity 0.6, scale 0.96, rotate -1deg
- 드롭 타겟: border-top 3px solid indigo

### 4-D. 반복 일정 아이템 (JS 동적 생성)

```
┌─────────────────────────────────────────────────┐
│ ⠿ ☑ 푸드플레이: 급여명세서                  [✕] │  ← 1줄
│    [매월] [10일] [정정교 ▼] 실행: 2026-03-09    │  ← 2줄
└─────────────────────────────────────────────────┘
```

- 클래스: `.ai-recurring-item`
- 2줄 레이아웃
  - 1줄: 드래그핸들(⠿) + 체크박스 + 제목 + 삭제(✕)
  - 2줄: 매월배지 + 날짜칩(클릭가능) + 담당자 select + 실행일
- 제목 더블클릭 → 인라인 input 편집 (`.ai-recurring-title`)
- 날짜칩 싱글클릭 → 인라인 select 변경 (`.ai-recurring-day`)
- 31일은 "말일"로 표시
- 담당자 드롭다운 변경 → 연결된 todo에도 반영
- 드래그 → todo 목록에 드롭하면 일반 할일로 변환
- 반대로 todo를 반복 영역에 드롭 → 날짜 선택 팝업 → 반복 등록

## 5. 상호작용 정리

### 토글 버튼
- 클릭 → 사이드바 열기/닫기
- 열림: 텍스트 "닫기", 본문 폭 줄어듦
- 닫힘: 텍스트 "AI" 또는 "할일"

### To-Do 카드
- **체크박스**: pending ↔ completed 토글
- **드래그핸들(⠿)**: 드래그앤드롭으로 순서 변경
- **반복 영역으로 드래그**: 날짜 선택 팝업 → 반복 등록
- **담당자 드롭다운**: 실시간 재할당
- **내용 더블클릭**: 인라인 편집 (input 변환, blur/Enter 저장, ESC 취소)
- **✕ 버튼**: 삭제

### 반복 일정 아이템
- **체크박스**: 이번 달 완료/미완료 토글
- **드래그핸들(⠿)**: todo 목록에 드롭 → 일반 할일로 변환 (반복 해제)
- **제목 더블클릭**: 인라인 편집
- **날짜칩 싱글클릭**: 인라인 select로 변환 → 날짜 변경
- **담당자 드롭다운**: 변경 시 연결된 todo에도 반영
- **✕ 버튼**: 반복 비활성화
- **접기/펼치기**: 헤더 클릭으로 토글, 접혀있어도 통계(미완료/완료) 표시

### 고객 말풍선 클릭
1. 텍스트 클립보드 복사
2. 메시지 분류 팝업 표시
3. To-Do/급여 저장 또는 삭제

### 상담원 말풍선 클릭
- 템플릿 저장 팝업

### AI 추천 답변 카드
- "입력창에 넣기" 클릭 → 채팅 입력란에 텍스트 삽입

### 드래그앤드롭 흐름
- **todo ↔ todo**: 순서 변경 (같은 목록 내)
- **todo → 반복 영역**: 반복 등록 (날짜 선택 팝업)
- **반복 → todo 목록**: 반복 해제 → 일반 할일 변환

## 6. CSS 레이아웃 요구사항 (핵심)

### 사이드바 본체 구조
```
#ai-sidebar (flex column, 100vh)
  ├── .ai-sidebar-header (flex-shrink: 0)
  ├── 필터 바 (flex-shrink: 0)
  ├── 입력 바 (flex-shrink: 0)
  ├── .ai-sidebar-body (flex: 1, overflow-y: auto, flex column)
  │   ├── #ai-status (상태 메시지)
  │   ├── #ai-recommendations (todo 목록)
  │   └── #ai-recurring-section (margin-top: auto → 하단 배치)
  │       ├── #ai-recurring-toggle (헤더 + 통계)
  │       └── #ai-recurring-list (max-height: 30vh, overflow scroll)
  └── .ai-sidebar-footer (flex-shrink: 0)
```

### 핵심 레이아웃 규칙
1. **반복 일정은 항상 하단**: `margin-top: auto`로 todo가 적어도 아래 유지
2. **반복 일정 최대 높이**: `max-height: 30vh` (화면 절반 이상 차지 방지)
3. **반복 일정 헤더는 큼직하게**: 16px 굵은 글씨, 26px 아이콘, todo 헤더와 동급
4. **접혀있어도 통계 보임**: 헤더에 "(N건 미완료 / N건 완료)" 항상 표시
5. **스크롤바**: 가느다란 커스텀 웹킷 스크롤바 (5-6px, 반투명)
